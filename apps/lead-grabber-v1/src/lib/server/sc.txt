### Conversational SMS Scenario Blueprints & Scoring Reference
================================================================

This document outlines 20 customer interaction scenarios. 
Each scenario includes the conversational path, the under-the-hood logic, 
and the corresponding impact on ClearSky engagement scores and intent buckets 
(cross-referenced with clearsky-scoring-reference.html and scoring.service.ts).

---

### Scenario 1: Standard Reschedule (Single Upcoming Event Found)

Marie: "Can I move my appointment to Monday?"
  │
  ▼
AI searches Google Calendar by phone for upcoming appointments.
  │
  ▼
One appointment found: Fri Jul 4 at 1:00 PM.
  │
  ▼
AI drafts: "You're booked Fri Jul 4 at 1pm. Pick a new time here: <link> — I'll cancel the old one once you rebook."
  │
  ▼
Marie clicks link (carries `?reschedule=eventId`) 
  │
  ▼
She picks a Monday slot 
  │
  ▼
New event created & Old event cancelled automatically.

[SCORING IMPACT]
- Orchestrator Webhook Score: +10 (Sales/Booking Intent)
- Pipeline Event Type: `sms_received` (+10 points) or `booking_rescheduled` (Conversion)
- Intent Bucket Progression: Promotes to [Active Project] (Score >= 45 & conversion event)

Pam leaves a voicemail: "Need to book a plumber for my sink." 
  │
  ▼
Pam texts: "Checking if you got my message?"
  │
  ▼
OpenAI semantic thread matching runs on recent inbound logs (within 7 days).
  │
  ▼
Orchestrator links the SMS and voicemail into the same conversation thread.
  │
  ▼
AI drafts: "Hi Pam! Yes, we received your voicemail about booking a plumber. You can pick an open time here: <link>"

### Scenario 17: User Pressed Wrong IVR Button (Reclassification)

  │
  ▼

  │
  ▼
Orchestrator detects mismatch: pressed category is "billing" but text content is "sales".
  │
  ▼
Orchestrator overrides digit routing and uses the "sales" path instead.
  │
  ▼
AI drafts: "Thanks for calling! You can book an estimate slot for your gutters right here: <link>"

[SCORING IMPACT]
- Orchestrator Webhook Score: +10 (Sales/Booking Intent override, not +0 Billing)

- Intent Bucket Progression: Promotes to [Comparison] or [Active Project]

### Scenario 18: Webhook Retry/Duplicate Suppression

Carrier network sends duplicate webhook for SMS event (already processing/processed).
  │
  ▼
Orchestrator checks `orchestrator_processed` metadata or searches `recentDrafts` matching `trigger_comm_id` within 10 mins.
  │
  ▼
Duplicate detected.
  │
  ▼
Orchestrator aborts immediately, preventing duplicate draft responses from being saved.

[SCORING IMPACT]
- Orchestrator Webhook Score: +0 (No-op)
- Pipeline Event Type: Suppressed (Identical content / Transaction ID match)
- Intent Bucket Progression: No change (Deduplication prevents double-scoring)

### Scenario 20: Completed Reschedule Loop (Booking Action Submit)

Marie picks a Monday 9am slot on the booking web page and clicks "Confirm Reschedule".
  │
  ▼
Booking Svelte action creates new event on Google Calendar.
  │
  ▼
Action reads the `reschedule` parameter containing the original event ID.
  │
  ▼
`deleteEvent` deletes the old Friday 1pm event from the calendar, sending a cancellation email automatically.

[SCORING IMPACT]
- Orchestrator Webhook Score: +10 (Sales/Booking Intent)
- Pipeline Event Type: `booking_submit` (Conversion event)
- Intent Bucket Progression: Promotes to [Active Project] (Score >= 45 & conversion event achieved)
